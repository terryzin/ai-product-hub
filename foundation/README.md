# foundation — 产品基础（深度分析）

这一层是 **AI Product Hub 自身**的产品基础定义，从 `raw/transcripts/` 的原始口述中**深度提炼**而来——不是复述表层功能清单，而是抽取背后的核心诉求、根因与范式转变。

> **贯穿三份文档的一句话主题：**
> 这不是"再做一个产品管理工具"，而是把产品管理从**旧范式**（面向人、靠人维护、用复制表达差异、状态分散不可知），重构为**新范式**（AI-native、由系统保证一致性、单一事实源 + 参数化变体 + profile 分支、全流程闭环且 agent 可读写）。

## 文档与分工

### A. 愿景层（从口述提炼）

| 文档 | 回答的问题 | 视角 |
|---|---|---|
| [用户画像.md](用户画像.md) | **为谁做？谁在协作？** | 人 + AI 的双层用户结构 |
| [产品痛点.md](产品痛点.md) | **旧范式为什么会崩？** | 从表象收敛到 5 大根因 |
| [产品需求.md](产品需求.md) | **新范式要具备什么能力？** | 从深层诉求推导能力需求 |

三者闭环：画像的**焦虑** → 痛点的**根因** → 需求的**对策**，可逐条对照追溯。

### B. 数据结构层（从现有 Excel 抽象）

从 `raw/extracted/` 的两份现有台账中抽象出“管了什么 + 需要什么数据结构”，作为平台数据模型设计输入：

| 文档 | 回答的问题 | 核心发现 |
|---|---|---|
| [数据结构-功能管理.md](数据结构-功能管理.md) | 功能表管了什么、需哪些结构 | 一张表挤了 5 个实体；6 变体列是四态多态值 | 
| [数据结构-资产管理.md](数据结构-资产管理.md) | 管哪些资产、需哪些结构 | 资产基类+多态子类；2 表其实非资产 |

两者汇到同一判断：**功能与资产共享同一套底层原语**（受控变体词表 + 适用性 + 版本 + 项目边 + Tag）。
