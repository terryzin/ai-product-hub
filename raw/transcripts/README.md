# raw/transcripts — 原始输入（录音转写）

产品设计的**源头**。这里只放最原始、未经加工的输入：录音转成的文字、口述记录、随手记的想法。后续所有 Feature、Spec、规划都从这里提炼。

## 在流水线里的位置

```
录音 → 转文字（放这里） → 提炼/拆解 → proposals/ 或 features/
  原始素材                  消化              结构化产出
```

- 这里的文件**保持原貌、只增不改**：原始记录就该留痕，方便回溯"当初到底怎么说的"。
- 真正的设计产出去 `features/`（一功能一文件）或 `proposals/`（待审议），不要在这里写 Spec。
- 一条输入消化完，在它头部把 `status` 标成 `已消化`，并用 `produces` 链接到产出。

## 命名

`YYYY-MM-DD-简短主题.md`，例如 `2026-06-21-主车数量讨论.md`。
同一天多条加序号：`2026-06-21-01-…`、`2026-06-21-02-…`。

## 每个文件的骨架

复制 [_模板.md](_模板.md) 开新文件。头部 YAML 是机器可读的元信息，正文贴转写全文：

| 字段 | 含义 |
|---|---|
| `source` | 来源：录音 / 口述 / 笔记 / 会议 |
| `date` | 录制或记录日期 |
| `title` | 简短主题 |
| `status` | 待消化 / 消化中 / 已消化 |
| `topics` | 涉及的区域或模块，便于检索（如 `[主车, 登录]`） |
| `produces` | 消化后产出的 feature/proposal 链接 |

## 状态流转

- `待消化` —— 刚转写完，还没处理。
- `消化中` —— 正在从中抽 Feature / 提议。
- `已消化` —— 已提炼完，`produces` 指向产出。
