## 转换约定（怎么读这些文件）

1. **表头自动识别**：源表存在 1~3 行表头（注释行 / 分组行 / 列名行）。规则=前 3 行里非空单元格最多的那行即真实列名行，记在 YAML 的 `header_row`。
2. **分组表头**：合并单元格的分组（如"基础信息""版本管理""合成数据"）记在 YAML 的 `groups`，并在字段字典的"分组"列体现。
3. **注释行**：列名行之上的整段注释（常含产品线定义、命名规范、基线规则）原样保留在 YAML 的 `note` 与正文"## 说明"。
4. **数据**：列名行以下的全部非空行，原样转为 Markdown 表；空单元格留空；日期归一为 `YYYY-MM-DD`。
5. **空列**：无标题的列记为 `(空列_X)`，保留以维持列对齐与保真。

## 已知保真限制

- **行高亮颜色未提取**：源表用"整行底色"表示是否纳入基线（绿=基线 / 蓝=未纳入）。该信息多由条件格式实现、openpyxl 读不到，故未转出。**请改用各表的版本/版别列**（如 `企业版⭐科研版`、`实现版本`）判断适用性——它们比行色更精确。
- **图片/缩略图未提取**：`缩略图/俯视图/截图` 等列在源表是嵌入图片，这里仅保留其单元格文本（通常为空）。
- **超链接仅取显示文本**：`相关文档链接` 等列的富文本超链接只保留可见文本。

## 如何重生成

源 Excel 更新后，用 openpyxl 重跑转换逻辑（前 3 行取非空最多者为表头、提取注释/分组、整表转 Markdown）覆盖本目录即可。转换依赖：`openpyxl`。
